Tak jak w poprzednich zadaniach domowych pracujemy na zbiorze danych HELOC. Wytrenujmy na nich model XGBoost. Jak widać z poniższej tabelki najbardziej znacząca jest zmienna ExternalRiskEstimate. Jest to zgodne z poprzednimi doświadczeniami, ponieważ samą tą zmienną można dość dobrze wytłumaczyć wynik (accuracy 0.70). Zauważmy również jak bardzo ta zmienna dominuje nad pozostałymi. Waga drugiej w kolejności zmiennej jest blisko 9-krotnie mniejsza niż pierwszej. Widzimy też że 6 zmiennych ma niedodatnią wartość, co znaczy że po usunięciu nich powinniśmy otrzymać lepszy model.

def avg(model, x, y, eq):
    pred = model.predict(x)
    acc = [1 if eq(e, s) else 0 for (e,s) in zip(pred, y)]
    return np.average(acc)

def filter_diff(x):
    rp = x['RiskPerformance']
    dis = len(set(rp))
    return dis == 1

def get_data():
    read = pd.read_csv("heloc_dataset_v1.csv")
    data_columns = [f for f in read.columns if f != 'RiskPerformance']
    grouped = read.groupby(data_columns)
    filtered = grouped.filter(lambda x: True)
    return filtered['RiskPerformance'], filtered[data_columns]

def prep_data():
    res, data = get_data()
    res = [1 if t == "Good" else 0 for t in res]
    for c in data.columns:
          data[c] = [-9 if t == -7 or t == -8 or t == -9 else t for t in data[c]]
    orig_data = data
    return orig_data, np.array(res), pd.DataFrame(data, columns=orig_data.columns)

    legend.append(name)
    imps = exp.importances
    l = dict()
    for i in imps:
        index = int(i.feature[1:])
        l[index] = i.weight
    allData.append(list([l[i] if i in l else 0 for i in range(0, len(varNames))]))
# take 7 with greatest sum of weights
sum = [0 for _ in varNames]
for d in allData:
    for i, v in enumerate(d):
        sum[i] = sum[i] + v
        
sorted = np.argsort(sum).tolist()
sorted.reverse()
sorted = sorted[0:7]
sorted.reverse()
allData = [[d[s] for s in sorted] for d in allData]
varNames = [varNames[s] for s in sorted]

res = pd.DataFrame({'rf1': allData[0], 'rf5': allData[1], 'xgb': allData[2]}, index = varNames)

Zauwazmy że w modelu używającym zmiennej ExternalRiskEstimate jest jedna istotna zmienna, podczas gdy w powyższych trzech modelach nieużywających tej zmennej są zawsze conajmniej trzy zmienne odgrywające znaczącą rolę, takie jak NetFractionRevolvingBurden, AverageMInFile czy PercentTradesNeverDelq. Wynika z tego że usunięcie tej zmiennej prowadzi do lepiej wyjaśnialnego modelu decyzji kredytowych. Wiadomo że zewnętrzny scoring zawsze odgrywa ważną rolę, podczas gdy dla nas istotniejsza jest informacja które konkretnie parametry sytuacji finansowej klienta są brane pod uwagę.

Zmienne pojawiające się jako istotne w powyższych modelach pojawiały się również w poprzednich pracach jako istotne, co potwierdza poprawność wyjaśnienia. Pomimo podobieństw między modelami, takich jak że zmienną z najwyższą wagą we wszystkich modelach jest NetFractionRevolvingBurden, występują też różnice między nimi. W modelu XGBoost drugą z najważniejszych zmiennych jest NumBank2NatlTradesWHighUtilization, podczas gdy w modelach lasu losowego ta zmienna jest odpowiednio na 6-tej.

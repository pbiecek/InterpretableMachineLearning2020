1) Key results and comments¶

Problem polega na predycji wydatkw na opiek zdrowotn osb fizycznych. Zbir danych zawiera dane pochodzce z sonday. Zmienne opisuj status demograficzny i spoeczno-ekonomiczny ankietowanych osb, jak rwnie diagnozy chorb, ograniczenia zdrowotne oraz zgoszony przez nich stan zdrowia.
Zbudowany model to Gradient Boosting Regressor, zaimplementowany w pakiecie scikit-learn. Zmienne kategoryczne (tj. zmienne, ktre maj co najwyej 10 unikalnych wartoci - nie dotyczy zmiennych "POVCAT15", "RTHLTH31", "MNHLTH31", ktre mog by traktowane jak cige) zostay przeksztacone za pomoc kodowania one-hot. Zmienna celu zostaa przeksztacona za pomoc logarytmu o podstawie 3.

ad 2)¶

Dla zbudowanego modelu obliczona zostaa permutacyjna wano zmiennych za pomoc funkcji permutation_importance() z pakietu scikit-learn. Boxploty dziesiciu najwaniejszych zmiennych zostay przedstawione na poniszym wykresie. Jak wida, zdecydowanie najwaniejsz zmienn jest zmienna INSCOV15, ktra oznacza wskanik zakresu ubezpieczenia zdrowotnego w 2015 roku i przyjmuje wartoci 1, 2, 3, ktre oznaczaj odpowiednio dowolne prywatne, tylko publiczne, nieubezpieczony. Kolejne, okoo dwukrotnie mniej wane zmienne to RACE3, HIBPDX (ktra oznacza diagnoz wysokiego cinienia krwi i przyjmuje wartoci -1, 1, 2, ktre oznaczaj odpowiednio niedotyczy, tak, nie), CHOLDX (ktra oznacza diagnoz wysokiego cholesterolu i przyjmuje wartoci -1, 1, 2, ktre oznaczaj odpowiednio niedotyczy, tak, nie).

ad 3)¶

Kolejne dwa modele zostay zbudowane podobnie jak bazowy. Rnic byo dodanie skalowania zmiennych liczbowych. W jednym modelu zosta uyty StandardScaler, natomiast w drugim MinMaxScaler.
Wyniki permutacyjnej wanoci zmiennych nie rni si pomidzy tymi dwoma skalowaniami, rni si natomiast w porwnaniu z bazowym modelem. Co prawda, nadal najwaniejsz zmienn jest INSCOV15, jednak w przypadku dodania skalowania jej wano zmalaa o okoo 0.01. Na kolejnych dwch miejscach pojawiy si zmienne, ktre wczeniej dla bazowego modelu nie wystpoway w najwaniejszej dziesitce: PCS42, AGE31X (ktra oznacza wiek pacjenta), a zmienna RACE3 spada z drugiego na czwarte miejsce. Z pozostaych zmiennych w rankingu poowa jest nowa, a druga poowa spada w porwnaniu z rankingiem dla bazowego modelu.
Wszystkie zmienne, ktre pojawiy si w najwaniejszej dziesitce, za wyjtkiem zmiennej PREGNT31 (ktra oznacza czy pacjent jest w ciy i przyjmuje wartoci -1, 1, 2, ktre oznaczaj odpowiednio niedotyczy, tak, nie) s zmiennymi cigymi. W rankingu bazowego modelu znajdoway si tylko zmienne kategoryczne.

Kolejne dwa modele to Random Forest Regressor oraz Linear Regression, oba ze skalowaniem StandardScaler.
Ranking dziesiciu najwaniejszych zmiennych dla modelu Random Forest Regressor zawiera wikszo zmiennych, ktre pojawiy si w rankingu dla modelu Gradient Boosting Regressor ze skalowaniem. Zmienne te wystpuj w nieco innej kolejnoci i daj wyniki innego rzdu (10 razy wiksze). W przypadku tego modelu najwaniejsz zmienn okazaa si zmienna AGE31X.
Ranking dziesiciu najwaniejszych zmiennych dla modelu regresji liniowej natomiast wyglda zupenie inaczej: wyniki s innego rzdu, wystpuj zmienne ktre nie wystpoway w poprzednich rankingach. Tutaj najwaniejsz zmienn okazaa si by zmienna EMPST31, ktra oznacza status zatrudnienia i przyjmuje wartoci -1, 1, 2, ktre oznaczaj odpowiednio nie dotyczy, zatrudniony, nie pracowa w okrelonym czasie ale ma prac do ktrej wrci po tym okrelonym czasie. Dla regresji liniowej wszystkie zmienne w rankingu s kategoryczne.

def transform_target_log3(data, target_name):
    val = data[target_name].values 
    return np.array([0 if v == 0 else np.log(v) / np.log(3) for v in val])


def get_categorical_features(data, not_cat_cols_list):
    num_unique = data.nunique()
    categorical_features = num_unique[num_unique <= 10].index.tolist()
    # Remove variables from categorical features list that can be treated as continuous
    for col in not_cat_cols_list:
        categorical_features.remove(col)
    return categorical_features


def print_model_results(y_train, y_pred_train, y_test, y_pred_test):
    mse_train = mean_squared_error(y_train, y_pred_train)
    rmse_train = np.sqrt(mse_train)
    
    mse_test = mean_squared_error(y_test, y_pred_test)
    rmse_test = np.sqrt(mse_test)
    
    print("Training: \nRMSE: ", rmse_train, " | MAE: ", mean_absolute_error(y_train, y_pred_train), " | R^2: ", r2_score(y_train, y_pred_train), "\n")
    print("Test: \nRMSE: ", rmse_test, " | MAE: ", mean_absolute_error(y_test, y_pred_test), " | R^2: ", r2_score(y_test, y_pred_test), "\n")


def calculate_pvi(model, X, y):
    return permutation_importance(model, X, y, n_repeats = 10, random_state = 123)


def plot_pvi_boxplots(pvi, n, data, model_name, img_path):
    pvi_sorted_idx = pvi.importances_mean.argsort()
    plt.figure(figsize = (10, 8))
    title = "Permutational variable importance for \n" + model_name
    plt.title(title, size = 14, fontweight = "bold")
    plt.grid()
    plt.boxplot(pvi.importances[pvi_sorted_idx][-n:].T, vert = False, labels = data.columns[pvi_sorted_idx][-n:])
    plt.xticks(fontsize = 12)
    plt.yticks(fontsize = 12)
    plt.savefig(img_path + ".png")
    plt.show()
